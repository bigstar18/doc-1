CentOS 5.3 + postfix + Dovecot + Extmail + Extman 邮件服务器配置  

2010-09-11 17:45:10|  分类： （新）服务器 |  标签： |举报 |字号大中小 订阅

       

用微信  “扫一扫”

将文章分享到朋友圈。

 

用易信  “扫一扫”

将文章分享到朋友圈。

 
下载LOFTER客户端

一直想配置一个接近EMOS的系统，因为他是用rpm 来安装的，方便管理，但一直没成功，目前暂时配置了一个基于rpm 和 yum 的安装，步骤不多，Maildrop 的工作由 postfix 来做了！呵呵~ 以后有时间而且技术成熟后再加上 Maildrop 和 防垃圾邮件（如 Spamassassin 或 DSPAM）、杀毒软件(如 clamav)。好，现在开始一个简单的邮件服务配置。

Postfix 2.3.3

邮件系统主程序

Dovecot

Pop3 及 IMAP 服务

Extmail 1.1.0

webmail 客户端

Extman 1.0.0

邮件管理平台

<为什么postfix 2.7都出了，还用2.3啊？因为我配置2.6的时候sasl 过不了，所以就用回旧的2.3！！这段时间没什么精力，所以没深入研究原因了，以后知道方法再告诉大家>

配置前先用yum 安装好配置过程中的依赖

# yum install rpm-build ## rpmbuild 依赖

# yum install db4-devel zlib-devel openldap-devel cyrus-sasl-devel pcre-devel mysql-devel openssl-devel gcc-c++ ## rpmbuild -ba 依赖

# yum install mysql-server ## mysql 的服务器安装

PS：以上安装包加上他们之间的依赖超过了40M，如查用管方源，可能要等很久，有两个方法可以供大家使用一，用原光盘iso 做源来安装，再update一下;二，用非官方源来安装，，速度都快很多的！两个方法都比较简单，相应的教程：

光驱做源 yum cdrom ：http://blog.163.com/buypc_hao/blog/static/10792077520092295819516/

定义yum的非官方库：http://blog.163.com/buypc_hao/blog/static/107920775200972895650137/ 

下载、安装 perl-Unix-Syslog、perl-GD 支持，extmail、extman 需要他们才能正常显示

下载地址：http://dag.wieers.com/rpm/packages/

# rpm -ivh perl-Unix-Syslog-1.0-1.el5.rf.i386.rpm

# rpm -ivh perl-GD-2.35-1.el5.rf.i386.rpm

把postfix 和 extmail 、 extman 先下载！放到一个习惯放软件包的文件夹中 ！

删除原来的sendmail 

# rpm -e --nodeps sendmail sendmail-cf

OK，开工咯~~~

# rpm -ivh postfix-2.3.3-2.src.rpm

# vi /usr/src/redhat/SPECS/postfix.spec

因为我需要mysql 认证postfix 的用户，所以修改一下postfix.spec ，

把%define MYSQL 0 改为 %define MYSQL 1

修改完成后建立一个二进制的postfix包

# rpmbuild -ba /usr/src/redhat/SPECS/postfix.spec

现在开始安装刚编辑过的postfix 包

# rpm -ivh /usr/src/redhat/RPMS/i386/postfix-2.3.3-2.i386.rpm

安装完成后，修改postfix的主配置文件mail.cf ，先把原来的主配置文件移开，新建立一个，把现成的配置贴进去（其实里面的sasl认证机制我还不会，所以只能直接复制别人的然后作个简单的修改）。

# mv /etc/postfix/main.cf /etc/postfix/main.cf.bak

# vi /etc/postfix/main.cf

#=====================BASE=========================

myhostname = mail.rookie.com

mydomain = rookie.com

myorigin = $mydomain

mydestination = $myhostname localhost localhost.$mydomain

mynetworks = 127.0.0.0/8, 10.44.100.0/24

inet_interfaces = all

#=====================Vritual Mailbox settings=========================

virtual_minimum_uid = 88

virtual_mailbox_base = /var/spool/mail

virtual_mailbox_maps = mysql:/etc/postfix/mysql_virtual_mailbox_maps.cf

virtual_mailbox_domains = mysql:/etc/postfix/mysql_virtual_domains_maps.cf

virtual_alias_domains =

virtual_alias_maps = mysql:/etc/postfix/mysql_virtual_alias_maps.cf

virtual_uid_maps = static:89

virtual_gid_maps = static:89

virtual_transport = virtual

maildrop_destination_recipient_limit = 1

maildrop_destination_concurrency_limit = 1

#====================QUOTA========================

message_size_limit = 52428800

mailbox_size_limit = 209715200

virtual_mailbox_limit = 209715200

virtual_create_maildirsize = yes

virtual_mailbox_extended = yes

virtual_mailbox_limit_maps = mysql:/etc/postfix/mysql_virtual_mailbox_limit_maps.cf

virtual_mailbox_limit_override = yes

virtual_maildir_limit_message = Sorry, the user's maildir has overdrawn his diskspace quota, please try again later.

virtual_overquota_bounce = yes

#====================SASL========================

broken_sasl_auth_clients = yes

smtpd_recipient_restrictions =

permit_mynetworks,

        permit_sasl_authenticated,

        reject_invalid_hostname,

        reject_non_fqdn_hostname,

        reject_unknown_sender_domain,

        reject_non_fqdn_sender,

        reject_non_fqdn_recipient,

        reject_unknown_recipient_domain,

        reject_unauth_pipelining,

        reject_unauth_destination,

        permit

smtpd_sasl_auth_enable = yes

smtpd_sasl_type = dovecot

smtpd_sasl_path = /var/run/dovecot/auth-client

smtpd_sasl_local_domain = $myhostname

smtpd_sasl_security_options = noanonymous

smtpd_sasl_application_name = smtpd

smtpd_banner=$myhostname ESMTP "Version not Available"

readme_directory = no

sample_directory = /etc/postfix

sendmail_path = /usr/sbin/sendmail

html_directory = no

setgid_group = postdrop

command_directory = /usr/sbin

manpage_directory = /usr/local/man

daemon_directory = /usr/libexec/postfix

newaliases_path = /usr/bin/newaliases

mailq_path = /usr/bin/mailq

queue_directory = /var/spool/postfix

mail_owner = postfix

添加4个和数据库有关的文件

# vim /etc/postfix/mysql_virtual_alias_maps.cf

user = extmail

password = extmail

hosts = localhost

dbname = extmail

table = alias

select_field = goto

where_field = address

additional_conditions = AND active = '1'

# vim /etc/postfix/mysql_virtual_domains_maps.cf

user = extmail

password = extmail

hosts = localhost

dbname = extmail

table = domain

select_field = description

where_field = domain

additional_conditions = AND active = '1'

# vim /etc/postfix/mysql_virtual_mailbox_limit_maps.cf

user = extmail

password = extmail

hosts = localhost

dbname = extmail

table = mailbox

select_field = quota

where_field = username

additional_conditions = AND active = '1'

# vim /etc/postfix/mysql_virtual_mailbox_maps.cf

user = extmail

password = extmail

hosts = localhost

dbname = extmail

table = mailbox

select_field = maildir

where_field = username

additional_conditions = AND active = '1'

postfix 的配置到这里就完成了。

Dovecot 配置

系统安装默认就装了Dovecot ，如果你的系统没装就自己yum  一个吧。

先移开原来dovecot 的配置文件

# mv /etc/dovecot.conf /etc/dovecot.conf.bak

再新建一个dovecot 配置文件

# vi /etc/dovecot/conf

base_dir=/var/run/dovecot

protocols=imap pop3

listen=*

disable_plaintext_auth = no

ssl_disable = yes

mail_location = maildir:/home/domains/%d/%n/Maildir

auth default {

mechanisms = PLAIN LOGIN CRAM-MD5 DIGEST-MD5

passdb sql {

args = /etc/dovecot-mysql.conf

}

userdb sql {

args = /etc/dovecot-mysql.conf

}

socket listen {

client {

path = /var/run/dovecot/auth-client

mode = 0660

user = postfix

group = postfix

}

}

}

first_valid_uid = 89

新建一个与mysql关联的文件

# vim /etc/dovecot-mysql.conf

driver = mysql

connect = host=/var/lib/mysql/mysql.sock dbname=extmail user=extmail password=extmail

default_pass_scheme = MD5

password_query = SELECT password FROM mailbox WHERE username = '%u'

user_query = SELECT maildir, 89 AS uid, 89 AS gid FROM mailbox WHERE username = '%u'

设置dovecot 开机自动启动

# chkconfig dovecot on

启动 dovecot 

# service dovecot start

配置extmail 和 extman

新建一个目录存放extmail 和 extman 

# mkdir /var/www/extsuite

解压下载extmail 和extman 包，然后移动到 /var/www/extsuite 中

# tar -zxvf extmail-1\[1\]\[1\].1.0.tar.gz

# tar -zxvf extman-1\[1\]\[1\].0.0.tar.gz

# mv extmail-1.1.0 /var/www/extsuite/extmail

# mv extman-1.0.0 /var/www/extsuite/extman

修改extmail 配置文件

# cp /var/www/extsuite/extmail/webmail.cf.default /var/www/extsuite/extmail/webmail.cf

# vi /var/www/extsuite/extmail/webmail.cf

SYS_SESS_DIR = /tmp 改为 SYS_SESS_DIR = /tmp/extmail

SYS_USER_LANG = en_US 改为 SYS_USER_LANG = zh_CN

SYS_USER_CHARSET = utf-8 改为 SYS_USER_CHARSET = gb2312

SYS_MYSQL_USER = db_user 改为 SYS_MYSQL_USER = extmail

SYS_MYSQL_PASS = db_user 改为 SYS_MYSQL_PASS = extmail

SYS_MYSQL_ATTR_NDQUOTA = netdiskquota改为SYS_MYSQL_ATTR_NDQUOTA = quota

SYS_G_ABOOK_FILE_CHARSET = utf-8改为SYS_G_ABOOK_FILE_CHARSET = gb2312

# vi /var/www/extsuite/extman/webman.cf

# SYS_LANG = en_US 改为 SYS_LANG = zh_CN

手动添加一行 SYS_CHARSET = gb2312

SYS_MYSQL_USER = webman 改为 SYS_MYSQL_USER = extmail

SYS_MYSQL_PASS = webman 改为 SYS_MYSQL_PASS = extmail

修改相应的权限

# chown -R postfix:postfix /var/www/extsuite/extmail/cgi

# chown -R postfix:postfix -R /var/www/extsuite/extman/cgi

# mkdir /tmp/extmail

# mkdir /tmp/extman

# chown -R postfix:postfix /tmp/extmail

# chown -R postfix:postfix /tmp/extman

# mkdir /home/domains

# chmod 775 domains

# chown :mail /home/domains (注意mail前有两点：少了这两点，chown root:mail 等于 chown :mail ，创建邮件用户后，用户是登录不上的，因为没有足够的权利新建文件夹)

配置Apache

# vim /etc/httpd/conf/httpd.conf

User apache 改为 User postfix

Group apache 改为 Group postfix

在最后的位置添加如下段

NameVirtualHost *:80

<VirtualHost *:80>

ServerName mail.rookie.com

DocumentRoot /var/www/extsuite/

ScriptAlias /extmail/cgi/ /var/www/extsuite/extmail/cgi/

Alias /extmail/ /var/www/extsuite/extmail/html/

ScriptAlias /extman/cgi/ /var/www/extsuite/extman/cgi/

Alias /extman/ /var/www/extsuite/extman/html/

</VirtualHost>

设置apache 开机自动启动

# chkconfig httpd on

启动apache 

# service httpd start

配置MySQL

默认的系统没有安装 默认没有密码，为了安全问题，大家可以通过mysqladmin -u root password 来自己设置一下密码

设置mysql 开机自动启动

# chkconfig mysqld on

启动mysql

# service mysqld start

导入extman 的相关数据

# mysql < /var/www/extsuite/extman/docs/extmail.sql

# mysql < /var/www/extsuite/extman/docs/init.sql

重启一下mysql

# service mysqld restart

进入mysql，授extmail 用户相应的权利，否则extman 新建用户和extmail 登录时会报权限不足

# mysql

mysql> grant all privileges on *.* to 'extmail'@'localhost' identified by 'extmail' with grant option;

Extman 管理员默认密码是 extmail*123*

如果登录不上，提示验证码出错，请查看服务器时间和要登录的物理机时间是否相差15分钟以上。

OK，配置完成前试了N久网上的教程，成功的是有，但有些还是不满意！以下是从wiki.extmail.org 中学习到的配置（夸一个，里面的教程是我所见过邮件服务器教程里，解释得最好的一个），以下教程基本和wiki 的一样，只是有一点点自己修改过，其中，他的平台是centOS4.7 的，我用的是最新的centOS 5.3 的！安装中使用了EMOS1.5做yum 库，使安装方便好多！注意，里面的部分包被修改后重新rpmbuild的！本教程自己在纯系统下试过两次，都成功滴，有不明白的可以在BLOG中留言。

再次注明，本教程修改于：http://wiki.extmail.org/extmail_solution_for_linux

一、安装前准备

1.安装时加上mysql server ，其它以默认安装（如果安装时忘记安装mysql，可以运行 yum install mysql-server 来安装）;

2.关闭防火墙;

3.关闭SELINUX

4.删除系统原来安好的sendmail  # rpm -e --nodeps sendmail sendmail-cf

5.vi 使用高亮 # vi ~/.bashrc 中在alias下面另上一行 alias vi='vim' 后重启（这一步可有可无）

6.建立EMOS1.5 的YUM源

# vi /etc/yum.repos.d/EMOS-Base.repo  #建立EMOS的库

# EMOS-Base.repo

#

# Created by ExtMail Dev Team: http://www.extmail.org/

#

# $Id$

[EMOS-base]

name=EMOS-Base

baseurl=http://mirror.extmail.org/yum/emos/1.5/os/$basearch/

gpgcheck=0

priority=0

protect=0

[EMOS-update]

name=EMOS-Updates

baseurl=http://mirror.extmail.org/yum/emos/1.5/updates/$basearch/

gpgcheck=0

priority=0

protect=0

# yum clean all 保存退出后清空原来的记录

二、配置MTA―postfix

# yum install postfix #安装postfix 包

# postconf -n >/etc/postfix/main2.cf

# mv /etc/postfix/main.cf /etc/postfix/main.cf.bak

# mv /etc/postfix/main2.cf /etc/postfix/main.cf

# vi /etc/postfix/main.cf #修改main.cf 配置文件

增加以下内容

# hostname

mynetworks = 127.0.0.1

myhostname = mail.rookie.com

mydestination = $mynetworks $myhostname

# banner

mail_name = Postfix - by extmail.org

smtpd_banner = $myhostname ESMTP $mail_name

# response immediately

smtpd_error_sleep_time = 0s

# Message and return code control

message_size_limit = 5242880

mailbox_size_limit = 5242880

show_user_unknown_table_name = no

# Queue lifetime control

bounce_queue_lifetime = 1d

maximal_queue_lifetime = 1d

三、配置courier-authlib

# yum install courier-authlib courier-authlib-mysql

修改authlib 配置文件

# rm -f /etc/authlib/authmysqlrc

# vi /etc/authlib/authmysqlrc

MYSQL_SERVER            localhost

MYSQL_USERNAME          extmail

MYSQL_PASSWORD          extmail

MYSQL_SOCKET            /var/lib/mysql/mysql.sock

MYSQL_PORT              3306

MYSQL_OPT               0

MYSQL_DATABASE          extmail

MYSQL_USER_TABLE        mailbox

MYSQL_CRYPT_PWFIELD     password

MYSQL_UID_FIELD         uidnumber

MYSQL_GID_FIELD         gidnumber

MYSQL_LOGIN_FIELD       username

MYSQL_HOME_FIELD        homedir

MYSQL_NAME_FIELD        name

MYSQL_MAILDIR_FIELD     maildir

MYSQL_QUOTA_FIELD       quota

MYSQL_SELECT_CLAUSE     SELECT username,password,"",uidnumber,gidnumber,\

                        CONCAT('/home/domains/',homedir),               \

                        CONCAT('/home/domains/',maildir),               \

                        quota,                                          \

                        name                                            \

                        FROM mailbox                                    \

                        WHERE username = '$(local_part)@$(domain)'

修改authmysqlrc 的权限和拥有者

# chown daemon.daemon /etc/authlib/authmysqlrc

# chmod 660 /etc/authlib/authmysqlrc

修改authdaemonrc 以下内容

authmodulelist="authmysql"

authmodulelistorig="authmysql"

启动

# service courier-authlib start

Starting Courier authentication services: authdaemond

修改authdaemon socket 目录权限，否则以下面的测试SMTP认证就不能通过

# chmod 755 /var/spool/authdaemon/

四、配置maildrop

# yum install maildrop

配置master.cf 为了使Postfix支持Maildrop，必须修改/etc/postfix/master.cf文件，注释掉原来的maildrop的配置内容，并改为：

maildrop   unix        -       n        n        -        -        pipe

  flags=DRhu user=vuser argv=maildrop -w 90 -d ${user}@${nexthop} ${recipient} ${user} ${extension} {nexthop}

配置main.cf 由于maildrop不支持一次接收多个收件人，因此必须在/etc/postfix/main.cf里增加如下参数：

maildrop_destination_recipient_limit = 1

测试maildrop对authlib支持

# maildrop -v

注意事项： 

1、如需重新编译Maildrop软件包，必须先获得其源码rpm包，并且必须先行安装courier-authlib及其devel软件包，否则编译后的maildrop将无法打开authlib支持。 

2、maildrop RPM包安装时，会自动创建vuser用户及vgroup用户组，专门用于邮件的存储，vuser:vgroup的uid/gid都是1000，这与一般的邮件文档中提及用postfix用户存邮件不一样。因为postfix用户的uid一般都低于500，而Suexec模块编译时对UID/GID的要求是要大于500，因此使用postfix用户不能满足要求。其次，如果用Maildrop作为投递代理（MDA），以postfix身份投递的话，会导致postfix MTA错误。

# Suexec config

    </VirtualHost>

# chkconfig httpd on

六、配置 extmail+extman

# mkdir /tmp/extman

# chown -R vuser:vgroup /tmp/extman/

由于RedHat发行版中包含了一个叫tmpwatch的工具，该工具会定期扫描/tmp/下的文件，如果这些文件很久都没被使用，将被删除，因此如果后台长期不使用，/tmp/extman目录有可能被tmpwatch删除，所以要么定期登陆后台，要么修改 webman.cf将临时目录修改到另一个地方。此处暂以/tmp/extman默认值为例。

# vi /var/www/extsuite/extman/docs/init.sql

把里面所有 extmail.org 的改为 rookie.com

# mysql </var/www/extsuite/extman/docs/extmail.sql

# mysql </var/www/extsuite/extman/docs/init.sql

上述导入初始化SQL时，默认的uidnumber/gidnumber都是1000，这和vuser:vgroup 的uid/gid一致，是因为maildrop投递时会从数据库里取uidnumber/gidnumber，而在master.cf里已经定义好了投递时的运行身份（vuser:vgroup），所以这两个字段的内容必须为1000，否则将出现投递错误，例如报0×06等错误。

八、配置courier-imap

默认的courier-authlib及courier-imap都会增加系统自启动设置，因此下一次服务器启动将自动启动相应的authlib及POP3服务

# yum install courier-imap

由于Courier-imap的IMAP目录是按UTF-7编码的，ExtMail目前还没有正式支持IMAP目录，因此需要屏蔽IMAP，只提供pop3服务。而就目前的使用情况来看，IMAP使用的非常少，绝大部分OutLook/Foxmail用户都习惯使用POP3而非IMAP。

# vi /usr/lib/courier-imap/etc/imapd

修改以下内容

    IMAPDSTART=NO

# vi /usr/lib/courier-imap/etc/imapd-ssl

修改以下内容

    IMAPDSSLSTART=NO
