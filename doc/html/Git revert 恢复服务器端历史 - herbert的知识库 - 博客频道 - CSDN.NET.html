如果一不把错误的代码Commit到本地，这时倒可以用Git reset HEAD^1 --hard来恢复到上个Commit，

可是如果已经把这个错误的代码错误的上传到服务器上，那么用git reset HEAD 就不管用了，因为再pull又会把错误的代码拿下来，

所以git revert 出场了， git revert + revision（如HEAD），就可以回退到HEAD之前的那个版本，并产生一个新的commit。
