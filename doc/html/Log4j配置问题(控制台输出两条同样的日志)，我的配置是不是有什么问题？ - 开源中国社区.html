Log4j配置问题(控制台输出两条同样的日志)，我的配置是不是有什么问题？

域名＋云解析经典组合，立减58元>>> »  

########################
#	Log4j配置
########################
log4j.rootLogger=WARN,console,f
log4j.logger.app=INFO,console
log4j.additivity.app=false
######################## 
# Rolling File 
########################
log4j.appender.f =org.apache.log4j.RollingFileAppender 
log4j.appender.f.File=../logs/log.log
log4j.appender.f.BufferSize=100
log4j.appender.f.MaxFileSize=2048KB 
log4j.appender.f.MaxBackupIndex=10 
log4j.appender.f.layout=org.apache.log4j.PatternLayout
log4j.appender.f.append=true 
log4j.appender.f.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss}  [CLASS]:%c[MESSAGE]:%m%n
################### 
# Console Appender 
################### 
log4j.appender.console=org.apache.log4j.ConsoleAppender
log4j.appender.console.Target=System.out
log4j.appender.console.layout=org.apache.log4j.PatternLayout
log4j.appender.console.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss}  [CLASS]:%c[MESSAGE]:%m%n

现在我的控制台总输出两条同样的日志，我想知道为什么？还有我的rootLogger的级别是WARN，控制台输出两条INFO级别的信息，没道理啊！

---------------问题补充---------------

@kylinhyl：我现在用上面的配置，现在控制台输出的信息有些奇怪，我第一次点submit的时候，控制台会输出两条同样的日志，当返回到结果页面时，我右键刷新时，只显示一条日志了。现象很奇怪啊(5年前)

    log4j.rootLogger=WARN,console,f

    log4j.logger.app=INFO,console

    你的appender定义了两遍，所以输出两次。去掉一个即可。

    级别以logger中显式定义的优先。
