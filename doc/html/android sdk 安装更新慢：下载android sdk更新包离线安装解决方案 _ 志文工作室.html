android sdk 安装更新慢：下载android sdk更新包离线安装解决方案

第一次安装android sdk后进行开发包的更新，你应该了解到需要的时间会很长，那么是否有办法提升安装的进度呢？这里提供一个取巧的方法，不会太麻烦，又能加快android开发环境的部署过程。具体请参考如下。

1、通过访问和分析android更新列表的xml在线地址，你可以获得最新完整的更新包下载信息，地址如下：
http://dl-ssl.google.com/android/repository/repository.xml

比如由此xml内容可分析到Android SDK Platform 3.2, revision 1的下载地址为：
http://dl-ssl.google.com/android/repository/android-3.2_r01-linux.zip

其他可类推获得，需要注意在2.2以前的版本是分操作系统平台的。下面是部分更新包下载地址：

2. 使用迅雷等P2P下载工具，复制获得的更新包下载地址进行下载。基于P2P工具共享下载原理，在你的网络畅通之前提下，下载速度会很快。

3. 在android sdk安装目录下查找Temp文件夹<如android-sdk-linux-x86/temp>，如果没有请创建。然后将下载所得的离线包放到Temp文件夹下(因为正常更新时会将更新包下载至这里)。

4. 以正常方式进行更新，你将会发现，它开始跳过Downloading 直接进行Installing了。
